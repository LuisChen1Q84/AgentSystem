# 语义记忆元数据规范

## 元数据头部格式

每个语义记忆文档必须包含以下元数据：

```yaml
---
INPUT: 输入/问题（这个知识回答什么）
OUTPUT: 输出/答案（这个知识包含什么）
POS: 位置/领域（属于哪个领域）
TAGS: 标签1, 标签2, 标签3（相关标签）
CREATED: 创建日期
UPDATED: 更新日期
---
```

## 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| INPUT | 是 | 描述这个知识回答什么问题 |
| OUTPUT | 是 | 描述这个知识包含什么内容 |
| POS | 是 | 位置/领域标签 |
| TAGS | 是 | 逗号分隔的标签 |
| CREATED | 是 | 创建日期 YYYY-MM-DD |
| UPDATED | 是 | 更新日期 YYYY-MM-DD |

## 示例

```yaml
---
INPUT: 什么是跨境支付
OUTPUT: 跨境支付的定义、模式、银联/网联角色
POS: 支付行业
TAGS: 跨境支付, 清算, 银联, 网联
CREATED: 2026-02-24
UPDATED: 2026-02-24
---
```

## AI检索使用

当需要检索语义记忆时：
1. 读取元数据头部的INPUT/OUTPUT/TAGS
2. 匹配用户需求与TAGS
3. 返回最相关的语义记忆
