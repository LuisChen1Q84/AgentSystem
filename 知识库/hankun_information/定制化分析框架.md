# 定制化分析框架
## 基于甲方业务场景的政策影响深度分析

> 版本：1.0
> 目标：提升政策分析从"通用框架"到"甲方定制"的深度

---

## 一、定制化分析的核心逻辑

### 1.1 为什么要定制化？

| 通用分析 | 定制化分析 |
|----------|------------|
| 政策对行业的影响 | 政策对甲方具体业务线的影响 |
| 行业合规要求 | 甲方现有合规体系的差距分析 |
| 竞争对手动态 | 甲方市场份额/竞争地位变化 |
| 风险预警 | 甲方具体的处罚风险敞口 |
| 建议方向 | 具体的落地方案+时间表+责任人 |

### 1.2 定制化的三个维度

```
                    ┌─────────────────┐
                    │   甲方业务场景   │
                    │  (业务线、产品)  │
                    └────────┬────────┘
                             │
        ┌────────────────────┼────────────────────┐
        │                    │                    │
        ▼                    ▼                    ▼
┌───────────────┐   ┌───────────────┐   ┌───────────────┐
│  业务影响度   │   │   合规差距    │   │   竞争格局    │
│  分析维度     │   │   分析维度    │   │   分析维度    │
└───────────────┘   └───────────────┘   └───────────────┘
        │                    │                    │
        └────────────────────┼────────────────────┘
                             │
                             ▼
                    ┌─────────────────┐
                    │   行动建议      │
                    │  (定制化方案)   │
                    └─────────────────┘
```

## 二、业务影响度分析维度

### 2.1 甲方业务矩阵

在分析政策前，需要明确甲方的业务结构：

| 业务线 | 产品/服务 | 收入占比 | 政策敏感度 |
|--------|-----------|----------|------------|
| 扫码支付 | 个人码/商户码 | 30% | 高 |
| 条码支付 | POS/线下收单 | 25% | 高 |
| 线上支付 | 电商/生活缴费 | 20% | 中 |
| 跨境支付 | 境外消费/汇款 | 15% | 高 |
| 金融产品 | 理财/保险 | 10% | 中 |

### 2.2 政策影响评估矩阵

对每项政策，评估对各业务线的影响：

| 政策要点 | 扫码支付 | 条码支付 | 线上支付 | 跨境支付 | 金融产品 |
|----------|----------|----------|----------|----------|----------|
| 限额调整 | -30%交易 | -20%交易 | -5%交易 | 0 | 0 |
| 费率管控 | -15%收入 | -10%收入 | -5%收入 | -20%收入 | 0 |
| 商户审核 | -10%商户 | -25%商户 | -5%商户 | 0 | 0 |
| 反洗钱 | -5%交易 | -15%交易 | -10%交易 | -30%交易 | -5% |

### 2.3 影响量化公式

```
业务收入影响 = Σ(政策强度 × 业务敏感度系数 × 业务收入基数)

合规成本 = 系统改造费用 + 人力成本 + 培训成本 + 外部顾问费

处罚风险敞口 = 违规概率 × 处罚上限 × 业务规模系数
```

## 三、合规差距分析维度

### 3.1 甲方合规体系自评

| 合规领域 | 甲方现状 | 政策要求 | 差距等级 |
|----------|----------|----------|----------|
| 反洗钱 | KYC+名单监控 | 人脸识别+实时监控 | ⚠️ 中 |
| 数据安全 | 本地存储 | 跨境传输限制 | ✓ 已满足 |
| 消费者保护 | 投诉处理 | 应急响应机制 | ⚠️ 小 |
| 备付金管理 | 100%交存 | 集中存管 | ✓ 已满足 |

### 3.2 差距整改优先级

```
优先级 = (合规风险等级) × (整改难度) × (业务重要性)

高优先级：风险高 + 整改易 + 业务重要
中优先级：风险高 + 整改难
低优先级：风险低
```

### 4.1 甲方市场份额追踪

| 业务领域 | 甲方份额 | 竞争对手A | 竞争对手B | 变化趋势 |

### 4.2 政策受益分析

## 六、使用方法

### 6.1 输入信息

在使用此框架前，需要收集甲方信息：

1. **业务清单**：各业务线、产品、服务
2. **收入结构**：各业务线收入占比
3. **合规现状**：现有制度、系统、人员
4. **竞争格局**：市场份额、主要对手
5. **战略目标**：短期/中期/长期目标

### 6.2 分析流程

```
1. 政策解读 → 2. 业务映射 → 3. 差距评估 → 4. 竞争研判 → 5. 方案定制
```
