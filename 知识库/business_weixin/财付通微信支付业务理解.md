## 二、业务架构

### 2.1 六大业务线

```
财付通业务矩阵
├── 扫码支付（商户收款码、个人收款码）
├── 条码支付（线下收单、POS）
├── 线上支付（电商、生活缴费）
├── 转账红包（社交支付）
├── 跨境支付（境外消费、跨境汇款）
└── 金融产品（理财通、保险）
```

### 2.2 收入模式

| 收入来源 | 说明 |
|----------|------|
| **交易手续费** | 按交易金额收取手续费（通常是0.6%-1%） |
| **增值服务** | 营销服务、数据服务 |
| **利息收入** | 备付金利息（集中存管后减少） |
| **跨境业务** | 跨境支付手续费 |

---

## 三、核心业务流程

### 3.1 扫码支付流程

```
用户扫码 → 商户确认 → 微信支付系统 → 网联/银联清算 → 商户结算
```

### 3.2 商户入驻流程

```
商户申请 → 资料审核 → 实名认证 → 签约入驻 → 开通支付 → 交易测试
```

### 3.3 资金流向

```
消费者 → 微信支付 → 网联/银联 → 财付通 → 商户账户（T+1结算）
```

## 四、监管合规要求

### 4.1 牌照资质

| 牌照类型 | 范围 | 状态 |
|----------|------|------|
| 互联网支付 | 全国 | 有效期至2026年 |
| 移动电话支付 | 全国 | 有效期至2026年 |
| 银行卡收单 | 全国 | 有效期至2026年 |

### 4.2 关键合规要求

| 领域 | 要求 | 关联制度 |
|------|------|----------|
| **备付金** | 100%集中存管 | 人民银行备付金管理办法 |
| **反洗钱** | KYC、名单监控、大额报告 | 反洗钱法、客户尽职调查办法 |
| **清算** | 必须通过网联/银联 | 断直连政策 |
| **消费者保护** | 投诉处理、资金安全 | 金融消费者权益保护实施办法 |
| **数据安全** | 个人信息保护 | 数据安全法、个人信息保护法 |

## 五、与政策相关的业务敏感点

### 5.1 高敏感业务

| 业务 | 政策敏感度 | 原因 |
|------|------------|------|
| **条码支付** | 极高 | 监管重点，限额管理 |
| **跨境支付** | 高 | 外汇管理、反洗钱 |
| **商户管理** | 高 | 违规处罚、商户审核 |
| **反洗钱** | 极高 | 处罚力度大 |

### 5.2 政策影响路径

```
政策变化 → 业务调整 → 系统改造 → 合规成本 → 收入影响
```

## 六、竞争格局

### 6.1 主要竞争对手

| 对手 | 市场份额 | 优势 |
|------|----------|------|
| **支付宝** | ~50% | 电商场景、余额宝 |
| **银联商务** | ~20% | 线下收单、POS |
| **京东支付** | ~5% | 电商场景 |

### 6.2 差异化竞争

| 维度 | 财付通优势 | 财付通劣势 |
|------|------------|------------|
| **用户基础** | 微信月活10亿+ | 境外用户少 |
| **场景** | 社交、小程序 | 大额交易 |
| **线下** | 二维码便捷 | POS布局少 |
| **跨境** | 起步晚 | 海外受理网络有限 |

## 八、政策分析要点

### 8.1 判断政策影响的三个维度

| 维度 | 问题 |
|------|------|
| **业务影响** | 影响哪块业务？交易量？收入？ |
| **合规成本** | 需要多大系统改造？多少人力？ |
| **竞争格局** | 对手是否受益？我们是否被动？ |

### 8.2 决策优先级

```
1. 合规红线（必须遵守）
   ↓
2. 收入影响（重点关注）
   ↓
3. 竞争变化（战略调整）
   ↓
4. 执行成本（资源配置）
```
