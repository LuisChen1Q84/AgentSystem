# PlugMem：插件式记忆框架

**来源**: 论文
**标签**: Memory、Plugin Architecture、Agent Framework

---

## 什么是 PlugMem

PlugMem 是一个**插件式记忆框架**，让 AI Agent 能够灵活地管理和利用不同类型的记忆。

核心创新：将不同类型的记忆（情景、语义、工作）整合为统一系统，并支持动态路由和可扩展插件架构。

---

## 核心概念

### 1. 多类型记忆

| 记忆类型 | 描述 |
|----------|------|
| **情景记忆 (Episodic)** | 具体事件和交互经历 |
| **语义记忆 (Semantic)** | 抽象知识和概念 |
| **工作记忆 (Working)** | 当前任务的临时信息 |

### 2. 记忆路由器 (Memory Router)

根据上下文动态决定使用哪些记忆：
- 分析当前任务需求
- 选择相关记忆源
- 组合为统一上下文

### 3. 记忆融合 (Memory Fusion)

将多个记忆源合并为连贯的上下文：
- 消除冲突
- 优先级排序
- 动态权重调整

## 关键方法论

### 1. 长期记忆更新

```
工作记忆 → 定期沉淀 → 长期记忆
```

定期将工作记忆中的重要信息转移到语义记忆。

### 2. 上下文感知的记忆检索

根据任务上下文动态调整检索策略：
- 任务类型
- 用户偏好
- 当前状态

### 3. 记忆优先级

动态计算记忆优先级：
- 相关性分数
- 时效性
- 使用频率
