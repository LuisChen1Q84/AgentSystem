# 第七阶段升级说明（自治经营与多代理协同）

## 目标

把系统从“会学习”升级为“可半自治经营”。

## 已落地能力

1. 自治目标引擎
- 脚本：`scripts/goal_autopilot.py`
- 产出：`日志/自治目标/YYYY-MM-DD.md`
- 策略文件：`目标系统/autopilot_goals.json`

2. 多代理协同层
- 脚本：`scripts/multi_agent_sync.py`
- 产出：`日志/多代理/YYYY-MM-DD.md`
- 状态文件：`目标系统/agent_state.json`

3. ROI 决策中枢
- 脚本：`scripts/roi_hub.py`
- 产出：`日志/ROI/YYYY-MM-DD.md`
- 输出：该做 / 延后 / 放弃

4. 实验验收自动化
- 脚本：`scripts/experiment_evaluator.py`
- 产出：`日志/实验评估/YYYY-MM-DD.md`
- 数据更新：`目标系统/experiments.json`

5. 发布控制与回滚建议
- 脚本：`scripts/release_controller.py`
- 产出：`日志/发布控制/YYYY-MM-DD.md`
- 状态文件：`目标系统/release_state.json`

6. 智能驾驶舱（CEO简报）
- 脚本：`scripts/ceo_brief_v2.py`
- 产出：`日志/CEO简报/YYYY-MM-DD.md`

7. 自治节奏编排
- 新增命令：`make cycle-autonomous`
- 执行链路：`cycle-evolve -> autopilot -> agents -> roi -> experiment-eval -> release-ctrl -> ceo-brief`

## 常用命令

```bash
make autopilot
make agents
make roi
make experiment-eval
make release-ctrl
make ceo-brief
make cycle-autonomous
```

## 价值

- 从“任务驱动”升级为“经营目标驱动”
- 将执行、审计、决策、发布纳入同一闭环
- 让 1 人公司具备准自治运营能力
