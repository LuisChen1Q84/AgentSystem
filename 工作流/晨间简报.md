# 晨间简报

**版本**: v1.0
**说明**: 每次进入系统自动展示今日待办和提醒

## 触发条件

- 用户进入AgentSystem目录
- 用户说"简报"、"晨报"
- 用户说"/morning"
- 每次新会话开始时

## 自动执行流程

```
1. 检查日期
2. 读取任务清单
3. 筛选今日待办
4. 检查已过期任务
5. 生成简报内容
6. 展示给用户
```

## 简报内容

```markdown
---

🌅 晨间简报 | 2026-02-24

## 📋 今日待办

### 【紧急重要】
- [ ] 任务1
- [ ] 任务2

### 【重要不紧急】
- [ ] 任务3

### 【日常事项】
- [ ] 任务4

## 📊 统计

- 总待办：4项
- 紧急重要：2项
- 重要不紧急：1项
- 日常事项：1项

## ⚠️ 已过期

- 已过期任务数量（如有）

## 📅 近期提醒

- 明天：任务X
- 后天：任务Y

## ❓ 你想从哪个任务开始？

---
```

## 今日待办筛选规则

### 紧急重要

- 截止日期 = 今天
- 用户标记为"紧急"

### 重要不紧急

- 截止日期 = 明天~下周
- 用户标记为"重要"

### 日常事项

- 无截止日期
- 用户标记为"日常"

## 自动任务检查

每次会话开始时：

1. **检查今日待办**：筛选截止日期为今天的任务
2. **检查已过期**：筛选截止日期<今天的未完成任务
3. **检查明日提醒**：筛选截止日期为明天的任务

## 已过期任务处理

当发现已过期任务时：

```
⚠️ 提醒：发现 X 个已过期任务

- 任务1（过期X天）
- 任务2（过期X天）

是否需要：
1. 延期处理
2. 取消任务
3. 保持原状
```

## 周日数据维护提醒

每周日额外显示：

```markdown
---

📊 周日数据维护提醒

本周需要完成：
1. 录入本周发布内容的数据（阅读量、互动量等）
2. 分析哪些内容表现好，为什么
3. 更新爆款递归库状态

是否现在开始？
```

```
- [ ] 更新知识库

```

### 示例2：有已过期任务

```
✓ 晨间简报已生成

【紧急重要】
- [ ] 完成政策分析报告

⚠️ 已过期：1项
- [ ] 更新支付行业数据（已过期3天）

❓ 你想从哪个任务开始？或者先处理过期任务？
```

## 强制规则

- 每次进入系统**必须**检查并显示简报
- 简报**必须**包含今日待办
- 发现已过期任务**必须**提醒用户
- 每周日**必须**显示数据维护提醒
