# 第二阶段升级说明（效率引擎）

## 目标

把系统从“稳定可用”升级到“高效产出”。

## 已落地能力

1. 增量索引与排序优化
- `scripts/knowledge_index.py build --mode incremental|full`
- 查询排序 = 相关性 + 时效 + 置信分

2. 任务自动拆解
- `scripts/task_planner.py split --task-id <TASK_ID>`
- 通过 `make task-plan id=<TASK_ID>` 一键执行

3. 智能下一步建议
- `scripts/recommender.py`
- 输出：`日志/建议/YYYY-MM-DD.md`

4. 内容流水线
- `scripts/content_pipeline.py --topic "..."`
- 输出：`产出/YYYY-MM-DD_<topic>.md`

5. 指标报表
- `scripts/metrics_report.py`
- 输出：`日志/指标/<date_range>.md`

## 常用命令

```bash
make index
make index-full
make recommend
make metrics
make pipeline topic="支付 监管 趋势"
make task-plan id="<TASK_ID>"
```
